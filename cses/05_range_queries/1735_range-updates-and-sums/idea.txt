Range Updates and Sums - CSES 1735
https://cses.fi/problemset/task/1735

## Đề bài
Cho một mảng gồm n phần tử. Cần xử lý q truy vấn thuộc ba loại:
  1 a b x : cộng x vào mọi phần tử trong đoạn [a, b]        (range add)
  2 a b x : gán giá trị x cho mọi phần tử trong đoạn [a, b]  (range assign)
  3 a b   : in ra tổng các phần tử trong đoạn [a, b]         (range sum)
Ràng buộc: 1 <= n, q <= 2*10^5 ; các giá trị và x nằm trong khoảng 1..10^6 ;
1 <= a <= b <= n.

## Ý tưởng
Dùng segment tree với lazy propagation, mỗi node lưu tổng của đoạn nó phủ.
Vì có ĐỒNG THỜI hai phép "gán đoạn" và "cộng đoạn", mỗi node cần HAI thẻ lazy
lồng nhau:
  - lz_a : giá trị assign đang chờ (sentinel NONE nghĩa là không có assign chờ).
  - lz_d : giá trị add đang chờ, được CỘNG SAU khi đã áp assign.
Ý nghĩa lazy của một node: nếu lz_a != NONE thì trước hết GÁN mọi phần tử bằng
lz_a rồi CỘNG thêm lz_d; nếu lz_a == NONE thì chỉ CỘNG lz_d.

Các phép áp lên một node phủ trọn trong đoạn truy vấn:
  - assign x : seg = x * (độ dài đoạn), đặt lz_a = x, lz_d = 0. Phép assign GHI ĐÈ
    toàn bộ lazy cũ bên dưới nên việc vứt bỏ lazy cũ là hợp lệ.
  - add x    : seg += x * (độ dài đoạn), lz_d += x (giữ nguyên lz_a nếu đang có).

Khi push lazy xuống hai con:
  - Nếu node có lz_a, áp phép gán lz_a xuống cả hai con trước.
  - Sau đó, nếu lz_d khác 0, áp riêng phép cộng lz_d xuống cả hai con.
Sau đó xóa lazy của node để giữ bất biến "seg luôn là tổng đúng của đoạn".

Mỗi truy vấn thu hẹp dần theo cây: push lazy dọc đường đi trước khi chạm các node
con, áp phép add/assign lên các node phủ trọn đoạn, rồi pull-up cập nhật lại tổng
cho các tổ tiên. Truy vấn tổng thì cộng seg của các node phủ trọn đoạn.

## Độ phức tạp
Mỗi truy vấn O(log n); tổng thể O((n + q) log n) thời gian và O(n) bộ nhớ.
Tổng có thể rất lớn (tới ~4*10^16) nên code dùng `long long`.
